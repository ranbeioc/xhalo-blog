Admin Scaffold

Stage 2.5 keeps the admin surface intentionally small. This page is a static operator-facing scaffold for verifying boundaries before a real Cloudflare Access protected dashboard exists.

Current shape

  • Static site stays on Cloudflare Pages.
  • Git remains the source of truth for posts and config.
  • Write paths are expected to open pull requests, not push to main.
  • Dynamic routes stay under explicit API and admin prefixes.

Scaffold APIs

  • GET /api/health
  • GET /api/scaffold
  • GET /api/posts
  • POST /api/tasks/example

Planned modules

Posts Index Read-only post metadata and publish state.
Draft Editor Draft creation that still publishes through GitHub pull requests.
Publishing Branch creation, PR creation, and Cloudflare Pages deployment status.
Moderation Comment and abuse review surfaces behind protected routes.

Boundary

This is not the production admin panel. It should not contain real secrets, production account identifiers, private content, or direct write flows to main.
