Admin Scaffold

Stage 3 begins with read-only platform primitives. This page is still a scaffold, but it now reflects the first D1-backed prototype slice before a real Cloudflare Access protected dashboard exists.

Operator guard

Secret required
Admin route access Cloudflare Access + request secret
Live writes disabled by default
Security verification (Turnstile)

Protected routes stay locked until the admin secret is provided. This is still an inner gate, not a substitute for Cloudflare Access.

Current shape

  • Static site stays on Cloudflare Pages.
  • Git remains the source of truth for posts and config.
  • Write paths are expected to open pull requests, not push to main.
  • Dynamic routes stay under explicit API and admin prefixes.
